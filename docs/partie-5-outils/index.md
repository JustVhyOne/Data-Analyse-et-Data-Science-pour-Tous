# 🛠 Partie 5 : Survivre dans le Monde Réel

> *"Le code, c'est bien. Le code qui ne casse pas votre ordinateur et que vous retrouvez dans 6 mois, c'est mieux."*

---

## 🎯 Pourquoi cette partie est cruciale

Vous avez appris à manipuler des données avec Pandas, à les visualiser avec Seaborn et Plotly, et même à entraîner un modèle de Machine Learning. **Félicitations !** 🎉

Mais il y a un dernier cap à franchir pour passer du statut de **"débutant enthousiaste"** à celui de **"praticien organisé"**.

Dans cette partie, nous allons aborder deux outils qui font la différence entre un projet qui tourne **un jour sur votre PC** et un projet **robuste, reproductible et partageable** :

- 📦 **L'environnement virtuel** : Pour éviter que l'installation d'une bibliothèque ne casse tous vos autres projets.
- 🐙 **Git & GitHub** : Pour sauvegarder votre code, collaborer, et montrer votre travail au monde entier.

Et bien sûr, tout cela reste **100% gratuit**.

---

## 🗺️ Ce qui vous attend

### 1️⃣ [L'Environnement Virtuel](environnement-virtuel.md)

Vous avez peut-être déjà vu des messages d'erreur bizarres en installant des bibliothèques Python. Souvent, c'est un **conflit de versions**.

Dans ce chapitre, vous apprendrez :

- 🤔 **Pourquoi** isoler ses projets est indispensable.
- 🐍 Utiliser **`venv`** (intégré à Python) pour créer un environnement propre.
- 📋 Gérer les dépendances avec un fichier `requirements.txt`.
- ☁️ Faire la même chose sur Google Colab (eh oui, c'est possible !).

!!! success "La promesse"
    Plus jamais vous ne direz *"Ça marchait hier, je ne comprends pas..."*.

### 2️⃣ [Git & GitHub Gratuit](git-github.md)

Git est le **couteau suisse du versionnement**. GitHub est la plateforme qui héberge vos projets et les rend accessibles au monde.

Dans ce chapitre, vous découvrirez :

- 🔄 Les bases de **Git** (`init`, `add`, `commit`, `push`).
- 🌐 Comment créer un dépôt **GitHub** gratuit.
- 📁 Mettre en ligne le site que vous êtes en train de lire.
- 🤝 Collaborer avec d'autres sans s'arracher les cheveux.

!!! tip "Bonus"
    Un profil GitHub actif est le **meilleur CV** pour un data scientist. Montrez ce que vous savez faire !
